Pentru cei ce nu vor sa foloseasca ATmelStudio:
1) Aflare cai executabile nevoite(avr-gcc,avrasm2,avr-obj,avr-dude).
    ATmelStudio>Tools>Options>Package configuration
    ex:E:\atmelstudio\7.0\toolchain\avr8\avr8-gnu-toolchain\bin\avr-gcc.exe
2) Legare cai de variabile de sistem(system variables)
    Edit system variables>Path:
      adaugare  E:\atmelstudio\7.0\toolchain\avr8\avr8-gnu-toolchain\bin ;compilator+avr-objcopy
                E:\atmelstudio\7.0\toolchain\avr8\avrassembler ;asamblor
                %USERPROFILE%\AppData\Local\Arduino15\packages\arduino\tools\avrdude\6.3.0-arduino17\bin\ ; incarcare/preluare cod de pe controller
                
    Edit system variables>Add:
         avr_libs:              E:\atmelstudio\7.0\toolchain\avr8\avr8-gnu-toolchain\avr\include ;biblioteci
         avr_pins:              E:\atmelstudio\7.0\toolchain\avr8\avr8-gnu-toolchain\avr\include\iom328p.h ;pentru a definii pinii
         avrddude_connf:        %USERPROFILE%/AppData/Local/Arduino15/packages/arduino/tools/avrdude/6.3.0-arduino17/etc/avrdude.conf
		 avrasm_include:		E:\atmelstudio\7.0\packs\atmel\ATmega_DFP\1.7.374\avrasm\inc ;pentru includerea datelor despre pinii specifici controller-ului
         Aceste variabile de sistem vor fi de folos pentru makefile sau pentru Intellisense folosit de VS Code
    Caile vor fi incluse in functie de locatia instalarii acelor executabile.
    OBS:Pentru unele versiuni de avrdude.conf procesul de compilare nu functioneaza.
        Pentru compilare este nevoie ca placa sa fie conectata la sistemul de calcul.
3) Instalare make:
	 Utilitarul make se poate procura prin Git BASH, Cygwin sau Chocolatey
4) Comunicare pe portul serial:
	 Se va instala PuTTY
     Edit system variables>Path:C:\Program Files\PuTTY
